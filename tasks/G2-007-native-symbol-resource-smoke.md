# G2-007 — Native Symbol Resource Smoke

Status: Proposed
Depends on: G2-006
Gate: G2
Type: HITL

## Goal

Prove that vector paths, named catalogs, explicit renderer registration, and one declared raster-icon
resource load and render through the real offscreen path in a GraalVM Native Image.

## Context

`mundane-map-native-tests` already builds `NativeSmokeMain` with metadata-repository use disabled.
G2-006 completes the symbol stack and portable JVM regression lane. Native-targeted code must not
discover symbol renderers or resources by scanning; any resource inclusion must be explicit and
reviewable in the native test module.

## Scope

- `modules/mundane-map-native-tests` smoke application, tests, resources, and native configuration.
- `modules/mundane-map-architecture-tests` checks for the exact resource declaration and continued
  rejection of resource or renderer discovery.
- A small raster-icon fixture with recorded origin/license when it is not generated by the test.
- Symbol-specific assertions added to the existing native smoke path.

## Out of scope

- General resource discovery, arbitrary image-format support, SVG, or a production resource scanner.
- Cross-platform Native Image claims or performance measurement.
- Moving Native Image execution into `qualityGate`.

## Acceptance criteria

- The JVM smoke and native executable construct a named catalog, install renderers explicitly, and
  render a vector path, composite symbol, and raster icon through `MapView`.
- The raster fixture is included by an explicit resource declaration or direct generated data; no
  classpath enumeration or implicit scanning is used.
- Smoke assertions verify stable non-background regions/bounds and at least one catalog lookup or
  registration diagnostic, rather than merely process exit.
- Native configuration is minimal, checked in, and documented at its point of use; metadata-agent
  output is not accepted without review.
- Architecture tests continue to reject prohibited dynamic runtime mechanisms.
- The maintainer completes the HITL checkpoint by providing a GraalVM Java 21 `native-image`
  toolchain and reviewing a successful `nativeSmoke` run and its resource configuration.

## Required tests

- JVM unit/smoke test for the complete symbol-resource code path.
- Architecture coverage for explicit registration and resource handling.
- Linux GraalVM Native Image build-and-run smoke using the same assertions.

## Validation

```bash
./gradlew :modules:mundane-map-native-tests:test :modules:mundane-map-architecture-tests:test --console=plain
./gradlew nativeSmoke --console=plain
./gradlew qualityGate --console=plain
git diff --check
```

## Notes

If GraalVM is unavailable, automated JVM work may be prepared but the task remains Proposed or becomes
Blocked with the missing external tool recorded; it cannot be marked Complete from JVM evidence
alone.
